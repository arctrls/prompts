# Cleanup Duplicates

Obsidian vault의 중복/준중복 문서를 탐지하고 상세 분석하여 정리 방안을 제안합니다.

## 필수 규칙

- 모든 파일 경로는 **절대경로**로 출력하세요.
- **파일 삭제나 수정은 사용자 확인 없이 절대 수행하지 마세요.**

## 작업 프로세스

1. `detect-duplicates` 도구를 호출하여 중복 그룹을 탐지합니다.
2. 각 중복 그룹에 대해:
   a. 마스터 문서와 중복 문서의 내용을 Read 도구로 읽습니다.
   b. 내용 차이점을 분석합니다 (단어 수, 태그, frontmatter, 고유 내용 등).
   c. 정리 방안을 제안합니다.
3. 종합 보고서를 작성합니다.

## 정리 방안 유형

- **병합**: 두 문서의 고유 내용을 합쳐 하나로 통합
- **삭제**: 완전 중복인 경우 사본 삭제
- **링크**: 관련되지만 중복이 아닌 경우 상호 링크 추가
- **유지**: 의도적 중복 (다른 컨텍스트)으로 판단되면 유지

## 출력 형식

```
## 중복 문서 분석 보고서

### 요약
- 탐지된 중복 그룹: N개
- 영향받는 문서: N개
- 예상 절감 문서 수: N개

### 중복 그룹 상세

#### 그룹 1: [주제/제목]
- **마스터**: /path/to/master.md (N단어, 태그 N개)
- **중복 1**: /path/to/dup1.md (유사도: 0.xx)
  - 차이점: [요약]
  - 권장 조치: [병합/삭제/링크/유지]
  - 사유: [설명]

### 실행 계획
1. [구체적 조치 1]
2. [구체적 조치 2]
- [실행하시겠습니까?]
```
